Git Workflow — AssetTrack (MDR)

github.com/luisInnovabyte/assettrack · Referencia rápida
main protegida PR obligatorio v1.0 · Marzo 2026
⛔ REGLA FUNDAMENTAL: Los becarios NUNCA hacen push directo a main. Todo cambio requiere un Pull Request aprobado por Luis.
🌿 Estructura de Ramas
main BLOQUEADA Producción — solo Luis via PR
angel/trabajo BECARIO Rama permanente de Ángel
francisco/trabajo BECARIO Rama permanente de Francisco
tarea-xxx LOCAL Temporal, se elimina al terminar
👨‍💻 Becarios — Flujo Diario día a día
1
Inicio de jornada — actualizar
git checkout angel/trabajo git pull --rebase origin main
2
Crear rama temporal para la tarea
git checkout -b tarea-login
3
Trabajar y hacer commits
git add . git commit -m "feat: descripcion"
4
Tarea lista — mergear en local
git checkout angel/trabajo git merge tarea-login git branch -d tarea-login
5
Subir rama a GitHub
git push origin angel/trabajo
6
Avisar a Luis
💬 "Tarea completada, rama lista para revisar"
🔄 Tras aprobación: git checkout angel/trabajo → git pull --rebase origin main
🔍 Supervisor (Luis) — Revisar y Aprobar PR review
1
Descargar rama del becario
git fetch origin git checkout angel/trabajo git pull origin angel/trabajo
2
Probar en local
Verificar funcionalidad manualmente
3
Merge en GitHub
GitHub → Pull Requests → angel/trabajo → Merge pull request [Squash and merge] ✓
4
Actualizar main local y avisar
git checkout main git pull origin main
⚡ Supervisor — Trabajo Propio acceso directo
1
Partir de main
git checkout main git pull origin main git checkout -b mejora-xxx
2
Mergear y publicar
git checkout main git merge mejora-xxx git branch -d mejora-xxx git push origin main
3
Propagar a ramas de becarios
git checkout angel/trabajo git merge main git push origin angel/trabajo git checkout francisco/trabajo git merge main git push origin francisco/trabajo
⚙️ Config. Inicial GitHub una sola vez
1
Crear ramas de becarios
git checkout main git checkout -b angel/trabajo git push origin angel/trabajo git checkout -b francisco/trabajo git push origin francisco/trabajo
2
Proteger main
Settings → Branches → Add rule Branch: main ✓ Require pull request ✓ Required approvals: 1
3
Pull Requests
Settings → General → PRs ✓ Allow merge commits ✓ Allow squash merging ✗ Auto-delete branches
